# Phase 16 — Compliance, Hardening, GA Launch

> **Duration:** 42–48 weeks (≈6 weeks focused) &nbsp; · &nbsp; **Owner:** Security Engineer + Compliance Lead &nbsp; · &nbsp; **Dependencies:** Phases 0–15
> **Companion:** [`../implementation-plan.md` §Phase 16](../implementation-plan.md)

---

## 1. Phase Objective & Business Purpose

Convert the engineered system into a *certified*, *pentested*, *documented*, *priced*, and *launched* product. Collect SOC 2 Type II evidence over a 6-month window (started at Phase 0), pass an external penetration test, finalize SDK and customer documentation, complete pricing, and execute GA.

**Business rationale:** every prior phase built the platform; Phase 16 monetizes it. SOC 2 + pentest evidence unlocks Enterprise revenue. Docs + SDKs unlock self-serve and developer adoption. Pricing + status page + on-call unlocks operational maturity. Without Phase 16, the platform is a science project.

## 4. Detailed Sub-Phases & Implementation Tasks

### 16.1 — SOC 2 Type II

Evidence collected over a continuous 6-month window. Categories:

- **Access reviews (quarterly):** auto-generated from `policy_audit` + `user_roles`. Every active user × role reviewed; managers attest.
- **Change management:** git history + ArgoCD deploys + PR review trail.
- **Incident response:** runbooks + actual incident records + post-mortems.
- **Vendor management:** SBOM (already generated by CI) + sub-processor list (in Trust page).
- **Security training:** tracked in HR system; mandatory annual.
- **Background checks** for employees with prod access.
- **Asset management:** inventory of services, data sources, dependencies.
- **Cryptography:** key inventory + rotation logs.
- **Vulnerability management:** CVE scan + remediation timeline.
- **Logging + monitoring:** Phase 0/5/15 deliverables become evidence.

Engage SOC 2 auditor 3 months before target date. Use Vanta / Drata / Secureframe to automate evidence collection (cheap; pays back tenfold during audit).

### 16.2 — ISO 27001 (Parallel)

Many controls overlap with SOC 2. Most expensive standalone: **information security risk assessment** — engage a consultant or use Vanta's framework.

Document the Information Security Management System (ISMS):
- Risk register.
- Asset inventory.
- Statement of Applicability (SoA).
- Internal audit cycle.

### 16.3 — HIPAA Readiness (Optional)

For healthcare prospects:
- Business Associate Agreement (BAA) template.
- PHI handling runbook.
- Encryption-at-rest with tenant-scoped CMKs (column-level envelope encryption on sensitive PG columns).
- Per-tenant data residency (Phase 15).
- Audit logs retained 6 years (existing WORM + per-tenant retention config).
- Workforce training records.

HIPAA is opt-in per tenant; charged as premium.

### 16.4 — External Penetration Test

Hire reputable firm (NCC Group, Cure53, Trail of Bits).

Provide:
- Source code (or representative samples).
- Dedicated test environment with realistic synthetic data.
- List of suspected weak points (honest disclosure).

Scope:
- Web app (admin console + customer-facing).
- API gateway.
- PG proxy (wire protocol, side channels, error leakage).
- AI orchestrator (prompt injection, exfiltration prompts).
- AuthN + step-up + break-glass.
- Multi-tenant isolation probes.
- DSL bounds (regex DoS, infinite loops).

Fix all critical + high findings before GA. Medium/low get tickets.

### 16.5 — Bug Bounty

Private program first (HackerOne / Intigriti) for 6 months; public program after 6 months of GA stability. Realistic bounties; SLA for triage.

### 16.6 — Customer Documentation

For every customer-facing surface:

- **Getting Started guide** (each persona: admin, steward, analyst).
- **Concept docs:** RBAC, ABAC, policies, retrieval, AI, risk, break-glass.
- **API reference:** auto-generated from OpenAPI.
- **SDK references:** Go, TS, Python (Phase 16 ships V1).
- **Runbooks (customer-side):** rotating keys, configuring webhooks, integrating SIEM, GDPR erasure.
- **Quickstarts:** "connect your first Postgres in 10 minutes," "author your first policy in 5 minutes."
- **Trust page:** sub-processors, certifications, security FAQ.
- **Changelog:** every release.

Docs hosted on `docs.platform.io` (Docusaurus / Mintlify / similar). Updated via PR.

### 16.7 — SDKs

V1 SDKs:
- **TS** (browser + Node.js): high priority for AI / web integrations.
- **Go**: backend developers integrating proxies / PDP.
- **Python**: data scientists + notebook integrations + AI.

Each SDK:
- Auto-generated from OpenAPI + hand-curated convenience wrappers.
- Webhook signature verification helpers.
- Examples for common flows.
- Versioned + published to package registries.

### 16.8 — Pricing & Packaging

Tiers:
- **Starter:** small teams, 1 tenant, 1 data source, 25 PG connections, no SAML, no advanced UEBA.
- **Pro:** SAML, multi-tenant, 100 connections, statistical UEBA V1, webhooks, audit retention 1 yr.
- **Enterprise:** dedicated PgBouncer, dedicated ClickHouse, on-prem LLM, HIPAA, custom retention, SSO + SCIM, BYO KMS.

Usage-based AI billing (markup over LLM cost; otherwise loss leader). Annual contracts for Enterprise. Free tier for design partners (V1 only).

Billing integration: Stripe (V1).

### 16.9 — Operational Maturity

- Status page (statuspage.io / Better Stack) live.
- Customer support tooling: Zendesk / Intercom integrated; routing by tier.
- On-call rotation with PagerDuty; tier-1 vs tier-2 differentiated.
- Customer onboarding flow tested by ≥ 3 design partners.

### 16.10 — Launch Day Operations

- Marketing site live with pricing.
- Press release / blog post / Hacker News.
- Design-partner case studies (2–3).
- "Day-one dashboard" for on-call (key SLOs, customer signups, errors).
- Slack #launch channel for cross-functional sync.
- Predefined incident command in case launch-day surge breaks something.

## 5. Architectural Gaps & Missing Requirements

1. **Tenant-scoped CMK encryption** for PG sensitive columns — design and implement before HIPAA tenants land.
2. **SCIM provisioning** for SSO tenants — most enterprises require.
3. **Data export per GDPR Article 20** — bulk export of a user's data + audit.
4. **Data-processing addendum** standardized in the contract.
5. **Sub-processor change-notification mechanism.**
6. **Customer-facing release notes** beyond changelog (highlights, migration guides).
7. **In-product upgrade prompts** for breaking SDK versions.
8. **Per-tenant SSO/MFA enforcement settings** in admin UI.
9. **Audit log SIEM exporter** standardized (CEF / LEEF / JSON over HTTPS).
10. **Customer success tooling** for usage tracking, expansion signals, churn risk.

| ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| Pentest finds a critical bug late                                 | Delay GA; transparent comms; fix + retest.                                                       |

| HackerOne triage SLA breach                                       | Dedicated triager + escalation path.                                                              |
| Pricing perceived as too high                                     | Free tier + design-partner pricing; iterate post-launch.                                          |
| SDK adoption low                                                  | Bounty for community contributions; office hours; SDK-of-the-month feature.                       |
| GA traffic exceeds capacity                                       | Phase 15 capacity model; staged enrollment if needed.                                              |
| Customer needs a feature not in V1                                | Roadmap transparency; design-partner program for prioritization.                                  |
| Compliance audit slips                                            | Communicate ETA; offer SOC 2 Type I bridge.                                                       |
